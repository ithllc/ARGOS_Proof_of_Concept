# Contributing to Mini-ARGOS POC

Thank you for your interest in contributing to the Mini-ARGOS Proof-of-Concept. This repository is provided under the GNU AGPL license. Contributions are welcome under the same license unless otherwise agreed in writing.

## Permitted Contributions

- Bug reports, feature requests, and reproducible examples are welcome via GitHub Issues.
- Small patches, improvements, and tests can be submitted via pull requests.
- Please make sure your contributions include tests where appropriate and follow the existing code style.

## How to Contribute

1. Fork the repository and create a feature branch.
2. Run tests locally and ensure they pass.
3. Create a pull request with a clear description of the change and why it is needed.

## Code of Conduct

Please be respectful and professional. If you need a formal Code of Conduct, we can add one; for now this project expects contributors to follow standard open-source community norms.
